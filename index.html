云数惠权益营销平台微信优惠券服务接入需求文档

1. 项目概述

1.1 项目背景

接入云数惠第三方微信优惠券服务商，封装其获取openid的h5、发券、数据回调、数据查询接口，为合作银行提供统一的优惠券发放服务，并在联合营销平台、银行联合营销平台展示相关数据。

1.2 项目目标

  • 与云数惠API对接，实现优惠券发放功能
  • 封装标准化接口供合作银行调用
  • 提供数据统计和查询功能
  • 建立完善的回调处理机制

2. 业务需求

3. 系统架构

3.1 系统组成

  • 云数惠接入层：对接云数惠API
  • 业务封装层：封装标准化接口
  • 数据存储层：存储发放和核销数据
  • 营销平台层：数据展示和权限管理
  • 银行营销平台：银行端数据查询

3.2 业务流程

3.2.1 发券流程

线下创建优惠券，优惠券信息填入联合营销平台——活动编码(营销平台提供即微信活动批次号)：batchCode；活动编码关联主题活动ID。

用户 → 接入方系统 → 我方系统 → 云数惠 → 微信

——使用活动编码+用户访问h5时获取的openid发放优惠券，并实时记录发放结果。

3.2.2 详细流程步骤

第一阶段：获取用户OpenID
  1. H5页面跳转：政府小程序跳转到云数惠H5页面（拉卡拉填中转地址）
  2. 微信授权：H5页面获取用户微信OpenID
  3. 回调返回：携带OpenID回跳到银行营销系统
    格式：${redirect_url}&openid=${response.data.openId}&from=dshy
第二阶段：优惠券发放
  1. API调用准备：银行营销系统准备发券参数（详细参数说明见技术需求章节）
  2. 发券请求：调用云数惠发放接口
  3. 云数惠处理：云数惠向微信平台发起优惠券发放
  4. 结果返回：微信平台返回发放结果给云数惠
  5. 同步保存：同步保存发放记录
第三阶段：回调与查询
  1. 核销回调：用户使用优惠券时，云数惠通过回调通知系统
  2. 数据查询：政府小程序查询发放结果
  3. 结果展示：银行营销系统返回查询结果

4. 技术需求

4.1 接口封装架构

接口封装设计：我方系统作为中间层，对外提供标准化接口给合作银行，对内对接云数惠API，实现参数转换、权限控制、数据记录等功能。

4.1.1 银行请求我方系统接口

接口名称：优惠券发放接口

接口用途：银行营销系统调用我方系统发放微信优惠券

请求参数：

  • bankId: 合作银行唯一标识，用于区分调用方银行，支持后续基于银行ID与主题活动ID的权限鉴权
  • openId: 微信用户唯一标识，通过H5授权页面获取（透传给云数惠）
  • batchCode: 活动编码/微信活动批次号，关联主题活动ID（透传给云数惠）
  • callTradeNo: 银行方交易流水号，用于幂等性控制（记录后关联，调用云数惠时生成我方系统流水号）
  • writeOffNotify: 银行方核销回调地址（记录后关联，调用云数惠时使用我方系统回调地址）

处理逻辑：

  1. 权限校验：验证bankId与batchCode关联的主题活动权限
  2. 参数校验：验证openId格式、batchCode有效性等
  3. 数据记录：记录银行方callTradeNo和writeOffNotify
  4. 流水号生成：生成我方系统交易流水号
  5. 调用云数惠：使用转换后的参数调用云数惠发放接口
  6. 结果处理：处理云数惠返回结果，记录发放明细

4.1.2 我方系统调用云数惠接口

接口名称：云数惠优惠券发放接口

接口用途：我方系统调用云数惠API发放微信优惠券

请求参数（向云数惠发送）：

  • openId: 微信用户唯一标识（从银行请求透传）
  • batchCode: 活动编码/微信活动批次号（从银行请求透传）
  • callTradeNo: 我方系统生成的交易流水号（非银行方流水号）
  • writeOffNotify: 我方系统的核销回调地址（非银行回调地址）
示例请求参数：
"openId": "oPa1U6a_nhfiR-5bdeZrnxzlx3_k",
"batchCode": "1017578611540230144",
"callTradeNo": "SYS1731749631383",
"writeOffNotify": "https://our-system.com/callback/writeoff"

4.1.3 回调处理接口（异步架构）

接口名称：核销回调处理接口

接口用途：接收云数惠的核销回调通知，采用异步架构处理银行通知

架构特点：接收云数惠回调后立即返回处理结果，后续异步处理合作银行通知逻辑

同步处理阶段：

  1. 接收回调：接收云数惠发送的核销通知
  2. 数据校验：校验回调数据完整性和合法性
  3. 立即响应：向云数惠返回处理确认，确保回调链路畅通

异步处理阶段：

  1. 数据解析：解析回调数据，提取核销信息和关联的原始请求记录
  2. 更新记录：更新我方系统的核销明细记录状态
  3. 参数转换：将云数惠回调参数转换为下游合作银行所需的请求格式
  4. 银行通知：根据原始请求记录中的bankId和writeOffNotify地址，异步向对应银行发送核销通知
  5. 通知重试：实现通知失败重试机制，确保银行能够接收到核销状态变更
  6. 日志记录：记录完整的回调处理和银行通知日志，便于问题排查

异步架构优势：

  • 响应速度：云数惠回调立即得到响应，避免超时重发
  • 系统解耦：云数惠回调处理与银行通知解耦，提高系统稳定性
  • 容错能力：银行系统异常不影响云数惠回调确认
  • 重试保障：异步重试机制确保银行最终能收到通知

4.1.4 数据查询接口封装

接口用途：为银行营销平台提供发放结果查询服务

查询范围：仅支持发放结果查询，核销数据基于回调机制获取，不提供查询接口

支持的查询接口：

  • 发放明细查询：根据银行ID、活动编码等条件查询发放记录和发放状态
  • 发放结果对账：调用云数惠查询接口进行数据对账，确保数据一致性

查询限制说明：

  • 核销数据：核销状态变更通过回调机制实时更新，不支持主动查询接口
  • 数据来源：查询数据来源于我方系统存储

权限控制：

  • 基于bankId进行数据权限隔离
  • 银行只能查询自己发放的优惠券数据
  • 后续支持基于银行ID与主题活动ID的权限鉴权

5.2 银行营销平台查询

银行营销平台需要提供完整的数据查询功能，支持银行查询相关的发放和核销数据，包含明细查询和汇总统计功能。

  • 微信优惠券发放明细查询：详细的优惠券发放记录查询
  • 微信优惠券核销明细查询：详细的优惠券核销记录查询

5.2.2 微信优惠券发放明细查询

原型链接：

微信优惠券发放明细查询.html

查询条件：

  • 活动编码：支持精确匹配查询
  • 优惠券类型：满减券/折扣券/代金券
  • 活动ID：活动UUID查询

  • 发券时间：支持时间范围查询

查询结果展示：

  • 流水号、活动ID、活动编码
  • OpenID、优惠券类型、优惠金额

  • 支持分页显示，每页可选5/10/20/50/100条
  • 支持多标签页管理

  • 活动ID：活动UUID查询

  • 支持分页显示和多标签页管理

5.2.4 查询功能技术特性

用户体验设计：

  • 多标签页管理：支持同时打开多个查询页面，提高操作效率
  • 智能分页：支持5/10/20/50/100条每页显示选项
  • 实时统计：查询结果显示总记录数和当前页范围
  • 响应式设计：适配不同屏幕尺寸的终端设备

查询性能优化：

  • 索引优化：基于查询字段建立数据库索引
  • 分页查询：避免大数据量查询影响系统性能
  • 缓存机制：热点查询数据缓存，提升查询速度
  • 异步加载：大数据量统计支持异步处理

数据安全控制：

  • 权限控制：基于银行机构的数据权限隔离
  • 操作日志：记录所有查询操作的审计日志
  • 敏感信息脱敏：OpenID等敏感信息部分脱敏显示
  • 导出限制：大批量数据导出需要审批流程
